INFO

SHIELD converts a standard USB drive into a physical kill switch. On insertion, it silently acquires elevated privileges, wipes the clipboard, terminates every running process in parallel, and forces an immediate shutdown via three redundant methods. No prompts. No warnings. No delay.

01 — Elevation
SILENT UAC BYPASS
Uses the Windows advapi32 token API to directly acquire SeShutdownPrivilege and SeDebugPrivilege. No dialog box. No user interaction. Falls back to a SYSTEM-level scheduled task if needed.
Read more →
02 — Termination
KILL EVERYTHING
Enumerates every running process and terminates them simultaneously across parallel threads. Maximum wait: 0.5 seconds. After that, shutdown fires regardless of whether all processes have exited.
Read more →
03 — Shutdown
TRIPLE REDUNDANCY
Three shutdown methods fire simultaneously: the Windows shutdown command, the InitiateSystemShutdownExW Win32 API, and a SYSTEM-level scheduled task. If one is blocked, the others execute.
Read more →

05
WRITE THE LISTENER
Manual Setup — Step 2
Paste the entire block shown into the Admin PowerShell window and press Enter. This writes the USB polling loop to disk at C:\Windows\Temp\shield_listener.ps1.

Verify with: Get-Content "C:\Windows\Temp\shield_listener.ps1" — output should begin with while ($true).
PowerShell (Admin)
PS> @'
while ($true) {
    $d = Get-WmiObject Win32_LogicalDisk |
      Where-Object { $_.VolumeName -eq "SHIELD" }
    foreach ($drive in $d) {
        $exe = $drive.DeviceID + "\shield.exe"
        if (Test-Path $exe) {
            Start-Process $exe -Verb RunAs
            exit
        }
    }
    Start-Sleep -Milliseconds 500
}
'@ | Set-Content -Path "C:\Windows\Temp\shield_listener.ps1" -Encoding UTF8
06
REGISTER THE TASK
Manual Setup — Step 3
Paste these four lines one at a time. The scheduled task runs the listener at every login with highest privileges.

On success you will see SHIELD_KillSwitch with state Ready or Running.

Note: Get-ScheduledTask may throw an XML error on retrieval — this is a Windows bug and does not affect functionality.
PowerShell (Admin)
PS> $action = New-ScheduledTaskAction -Execute "powershell.exe" -Argument '-WindowStyle Hidden -NonInteractive -ExecutionPolicy Bypass -File "C:\Windows\Temp\shield_listener.ps1"'
PS> $trigger = New-ScheduledTaskTrigger -AtLogOn
PS> $settings = New-ScheduledTaskSettingsSet -Hidden -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -ExecutionTimeLimit 0
PS> Register-ScheduledTask -TaskName "SHIELD_KillSwitch" -Action $action -Trigger $trigger -Settings $settings -RunLevel Highest -Force

Code Structure

Inside killswitch.py

Select any function for a detailed breakdown.

elevate_privileges()
-> bool
Silent UAC bypass via advapi32 token manipulation. Enables SeShutdownPrivilege and SeDebugPrivilege. Falls back to SYSTEM scheduled task on failure.
Silent elevation
_enable_privilege()
h_token, name -> void
Internal helper. Defines Win32 LUID structs via ctypes and calls AdjustTokenPrivileges to enable the named privilege on the given token handle.
Win32 API
_scheduled_task_bypass()
-> bool · fallback
Registers a one-shot scheduled task running as SYSTEM. Fires shutdown /s /f /t 0 immediately and deletes itself after execution.
Fallback
clear_clipboard()
-> void
Direct Win32 OpenClipboard() and EmptyClipboard() call via ctypes. Faster than spawning a PowerShell process. No external dependencies.
Win32 direct
get_all_processes()
-> list[(pid, name)]
Runs tasklist /fo csv with CREATE_NO_WINDOW flag. Parses output into (PID, name) tuples representing every process at trigger time.
Enumeration
kill_all_processes()
-> void
Spawns one thread per process and fires taskkill /F /PID on all simultaneously. 0.5s max thread join wait. Then returns regardless.
Parallel kill
force_shutdown()
-> void · no return
Fires three methods simultaneously: shutdown command, InitiateSystemShutdownExW Win32 API, and a SYSTEM scheduled task nuclear fallback.
Triple method
run_shield()
-> void · conductor
Master function called from __main__. Orchestrates the full sequence and sleeps 10s at end to keep the process alive for shutdown to complete.
Entry point
log_event()
message: str -> void
Writes timestamped entries to shield_log.txt. Always wrapped in try/except — a logging failure never blocks the shutdown sequence.

Responsible Use

White-Hat Principles

NOTE

Publishing security tooling is standard practice in the infosec community. What separates white-hat from misuse is intent, documentation, and scope. SHIELD is built for systems you own and authorised to protect.

PERMITTED
  • Deploy on devices you own or have explicit written authorisation to protect
  • Publish source code only — never distribute compiled executables
  • Document intent clearly in README and licence
  • Test in a controlled, isolated environment before deployment
  • Pursue certifications: CEH, CompTIA Security+, OSCP
  • Engage the security community via CTFs and bug bounty programmes
PROHIBITED
  • Deploy on any machine you do not own without written permission
  • Distribute pre-compiled .exe files of this or any security tool
  • Modify to trigger on any USB — the SHIELD label is the safety gate
  • Use to cause disruption, data loss, or denial of service
  • Deploy on workplace machines without explicit IT authorisation
  • Remove or alter the ethical disclaimer in public distributions
